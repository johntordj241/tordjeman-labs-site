## Cahier des Charges — Site Tordjeman Labs (Version 2.0)

### 1. Positionnement
- Le site public est une vitrine institutionnelle figée.
- Tordjeman Labs est un hub stratégique d’orchestration (recherche, cadrage, gouvernance, écosystèmes).
- Le hub est piloté hors site, via TAI-Systeme (Notion), source éditoriale unique.

### 2. Architecture
- **Site public** : lecture seule, aucune authentification, aucune logique métier.
- **Pilotage éditorial** : exclusif via Notion (TAI-Systeme) ; le site n’interprète rien.
- **Hub privé** : hors site, invisible, non accessible publiquement.

### 3. Pages autorisées (liste blanche)
1. Accueil
2. À propos / Le Laboratoire
3. Expertises / Axes
4. Méthodologie
5. Recherche & Prospective
6. Gouvernance & Cadres éthiques
7. Publications (lecture seule)
8. Modalités de collaboration
9. Contact institutionnel

Toute autre page est interdite.

### 4. Fonctionnalités autorisées
- Lecture de contenu statique.
- Téléchargement de documents publics validés.
- Animation du logo : **page d’accueil**, **section hero**, **1 fois par session**.
- Rituel d’entrée audiovisuel (documenté ci-dessous) : animation + micro-signature sonore lors du premier chargement de session, sans répétition.

### 5. Fonctionnalités interdites
- Administration web, CMS embarqué, login.
- Chatbot public, IA visible, paiement, KPI, forum/commentaires.
- Tracking marketing agressif ou collecte intrusive.

### 6. Gouvernance éditoriale
- Toutes les modifications passent par Notion (TAI-Systeme).
- Le site lit uniquement les éléments « Statut = Publié » et « Niveau = Public ».
- Le site ne prend aucune décision : il reflète le contenu autorisé.

### 7. Niveau d’intervention
- Public : vitrine actuelle, contenu validé, lecture seule.
- Privé : atelier interne, accès nominatif, traçabilité, hors dépôt.
- Confidentiel : accompagnements spécifiques, clauses juridiques dédiées, non exposé.

### 8. Règle de gel
- Toute évolution nécessite une décision stratégique formelle documentée.
- Sans nouveau mandat, le site reste inchangé.

### 9. Référence
- Ce document remplace tout cahier des charges antérieur.
- Aucune interprétation n’est autorisée hors du périmètre ci-dessus.

### 10. Rituel d’entrée audiovisuel (documentation)
- Existence reconnue d’un rituel combinant animation du logo et micro-signature sonore.
- Déclenchement : uniquement sur la page d’accueil, lors du premier chargement de session.
- Pas de répétition, pas d’interaction, pas de diffusion sur d’autres pages.
- Cet actif identitaire est documenté mais **non activé par défaut** dans le code. Toute exploitation future exige une décision stratégique formelle.
